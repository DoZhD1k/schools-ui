# API Документация: Модуль "Рейтинг Школ"

## 1. Общая информация

**Base URL:** `https://admin.smartalmaty.kz/api/v1/institutions-monitoring/`

Все запросы к защищенным эндпоинтам должны содержать заголовок `Authorization`.

**Пример заголовка:**

```
Authorization: Token 93f28f3a-b860-4801-9197-3cfc50...
```

## 2. Аутентификация

### Получение токена аутентификации

Для начала работы пользователю необходимо получить токен, отправив свои учетные данные.

`POST /school-rating/login/`

**Тело запроса (Request Body):**

```json
{
  "email": "admin@test.com",
  "password": "password123"
}
```

**Ответ при успехе (200 OK):**

```json
{
  "token": "93f28f3a-b860-4801-9197-3cfc50a5f6e8"
}
```

**Ответ при ошибке (401 Unauthorized):**

```json
{
  "error": "Неверные учетные данные"
}
```

## 3. Администрирование (Только для роли "Администратор")

Эти эндпоинты предназначены для управления пользователями и ролями. Доступ к ним имеет только пользователь с ролью "Администратор".

### 3.1. Управление пользователями

#### Получение списка всех пользователей

`GET /school-rating/admin/users/`

Возвращает список всех зарегистрированных пользователей в системе.

**Ответ при успехе (200 OK):**

```json
[
  {
    "id": 1,
    "email": "admin@test.com",
    "first_name": "Админ",
    "last_name": "Админов",
    "patronymic": "",
    "birth_date": null,
    "position": "Главный администратор",
    "role": 1,
    "school": null,
    "role_name": "Администратор",
    "school_name": null,
    "is_active": true
  },
  {
    "id": 2,
    "email": "school_a@test.com",
    "first_name": "Учитель",
    "last_name": "Школы А",
    "patronymic": "Иванович",
    "birth_date": "1990-05-15",
    "position": "Директор",
    "role": 2,
    "school": 19663,
    "role_name": "Организация образования",
    "school_name": "Тестовая Школа А",
    "is_active": true
  }
]
```

#### Создание нового пользователя

`POST /school-rating/admin/users/`

Создает новую учетную запись пользователя.

**Тело запроса (Request Body):**

```json
{
  "email": "new_user@test.com",
  "password": "password123",
  "first_name": "Новый",
  "last_name": "Пользователь",
  "patronymic": "Петрович",
  "birth_date": "1985-11-20", // Формат: YYYY-MM-DD
  "position": "Директор",
  "role": 2, // ID роли из эндпоинта /admin/roles/
  "school": 19663 // ID школы из эндпоинта /data/schools/, null для администраторов
}
```

**Ответ при успехе (201 Created):**
_Тело ответа будет содержать объект созданного пользователя (аналогично объекту из списка)._

#### Получение, обновление и удаление пользователя

- `GET /school-rating/admin/users/{id}/` - Получить информацию о конкретном пользователе.
- `PUT /school-rating/admin/users/{id}/` - Полностью обновить информацию о пользователе.
- `PATCH /school-rating/admin/users/{id}/` - Частично обновить информацию о пользователе.
- `DELETE /school-rating/admin/users/{id}/` - Удалить пользователя.

**Пример тела запроса для `PATCH`:**

```json
{
  "first_name": "Иван",
  "is_active": false
}
```

### 3.2. Управление ролями

#### Получение списка ролей

`GET /school-rating/admin/roles/`

Возвращает список всех доступных ролей в системе. Используется для заполнения выпадающего списка при создании/редактировании пользователя.

## 4. Работа с данными

Эти эндпоинты предназначены для просмотра и редактирования данных (успеваемость, безопасность и т.д.). Права доступа зависят от роли пользователя.

- **Администратор / Управление образования:** Видят и могут редактировать данные всех школ.
- **Организация образования:** Видят и могут редактировать данные **только своей школы**.

### 4.1. Общий шаблон для всех эндпоинтов данных

Все эндпоинты данных (`/schools/`, `/academic-performance-22-23/` и т.д.) работают по единому принципу.

#### Получение списка записей

`GET /{resource_name}/`

- **Для Администратора:** Вернет полный список записей.
- **Для Пользователя школы:** Вернет **только те записи**, которые относятся к его школе.

#### Создание новой записи

`POST /{resource_name}/`

- **Для Администратора:** Необходимо явно указать `school` (ID школы), к которой относится запись.
  ```json
  {
    "school": 19663,
    "grade_level_5": 25,
    "grade_level_4": 40
    // ... другие поля
  }
  ```
- **Для Пользователя школы:** Поле `school` указывать **не нужно**. Оно будет установлено автоматически на сервере.
  ```json
  {
    "grade_level_5": 15,
    "grade_level_4": 30
    // ... другие поля
  }
  ```

#### Обновление и удаление записей

- `PUT /data/{resource_name}/{id}/`
- `PATCH /data/{resource_name}/{id}/`
- `DELETE /data/{resource_name}/{id}/`

Пользователь с ролью "Организация образования" сможет выполнить эти операции только над записями, принадлежащими его школе. Попытка изменить чужую запись вернет ошибку `404 Not Found` или `403 Forbidden`.

## 5. Коды ошибок

- `401 Unauthorized`: Запрос без токена или с неверным токеном. Необходимо заново получить токен через `/login/`.
- `403 Forbidden`: Пользователь аутентифицирован, но у его роли нет прав на выполнение данного действия (например, пользователь школы пытается получить доступ к `/admin/users/`).
- `404 Not Found`: Запрашиваемый ресурс не найден. Для пользователей школ это также может означать, что ресурс существует, но относится к другой школе.
- `400 Bad Request`: Ошибка в данных, отправленных в теле запроса (например, пропущено обязательное поле). Ответ будет содержать описание ошибки.
