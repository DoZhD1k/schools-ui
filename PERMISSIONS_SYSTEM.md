# Система разрешений и ролей

## 🎯 Обзор

Реализована полная система разграничения доступа на основе ролей пользователей, которая работает исключительно с данными из API.

## 📊 Роли пользователей

### 1. Администратор (ID: 1)

- **Область доступа**: Полный доступ ко всем данным и функциям
- **Разрешения**: Все разрешения определяются из API
- **Особенности**:
  - Управление пользователями и ролями
  - Доступ к системным настройкам
  - Просмотр и управление всеми школами

### 2. Организация образования (ID: 2)

- **Область доступа**: Только данные своей школы
- **Разрешения**: Определяются из API (обычно ввод данных, просмотр отчетов)
- **Особенности**:
  - Видит только свою школу
  - Может изменять только данные своей школы
  - Ограниченный доступ к аналитике

### 3. Управление образования (ID: 3)

- **Область доступа**: Все школы в регионе
- **Разрешения**: Определяются из API (обычно все кроме админских функций)
- **Особенности**:
  - Видит все школы
  - Может просматривать аналитику и отчеты
  - Не может управлять пользователями

## 🛠 Архитектура системы

### Ключевые файлы:

1. **`lib/role-permissions.ts`**

   - Определение типов ролей и разрешений
   - Класс `PermissionChecker` для проверки прав доступа
   - Карта областей доступа к данным

2. **`hooks/usePermissions.ts`**

   - Хуки для работы с разрешениями
   - Фильтрация данных по правам доступа
   - Интеграция с auth context

3. **`components/auth/role-guard.tsx`**

   - Компоненты-охранники для ограничения доступа
   - Специализированные guards для разных функций

4. **`contexts/auth-context.tsx`**
   - Интеграция разрешений с системой авторизации
   - Передача данных о роли пользователя

## 🎨 Использование

### Проверка разрешений в компонентах

```tsx
import { usePermissions, useCanAccess } from "@/hooks/usePermissions";

function MyComponent() {
  const permissions = usePermissions();
  const canViewAnalytics = useCanAccess("analytics");

  if (!permissions?.canInputData()) {
    return <div>Нет прав для ввода данных</div>;
  }

  return (
    <div>
      {canViewAnalytics && <AnalyticsButton />}
      {permissions.canManageUsers() && <AdminPanel />}
    </div>
  );
}
```

### Использование Guards

```tsx
import { AdminGuard, AnalyticsGuard } from "@/components/auth/role-guard";

function App() {
  return (
    <div>
      <AdminGuard fallback={<div>Нет доступа</div>}>
        <AdminPanel />
      </AdminGuard>

      <AnalyticsGuard>
        <AnalyticsCharts />
      </AnalyticsGuard>
    </div>
  );
}
```

### Фильтрация данных

```tsx
import { useDataFilter } from "@/hooks/usePermissions";

function SchoolsList({ schools }) {
  const { filterSchoolData } = useDataFilter();

  // Автоматически фильтрует школы по правам доступа
  const accessibleSchools = filterSchoolData(schools);

  return (
    <div>
      {accessibleSchools.map((school) => (
        <SchoolCard key={school.id} school={school} />
      ))}
    </div>
  );
}
```

## 🔧 Принципы работы

### 1. Источник данных о разрешениях

- **Приоритет**: Данные из API (`rolePermissions`)
- **Fallback**: Базовые значения по умолчанию только для типизации
- **Логика бизнеса**: Области доступа определяются на клиенте

### 2. Области доступа к данным

- `ADMIN_FULL`: Администратор - полный доступ
- `ALL_SCHOOLS`: Управление образования - все школы
- `OWN_SCHOOL_ONLY`: Организация образования - только своя школа

### 3. Проверка разрешений

```typescript
// Базовые разрешения из API
permissions.canInputData(); // Ввод данных
permissions.canViewReports(); // Просмотр отчетов
permissions.canViewAnalytics(); // Просмотр аналитики
permissions.canFormRating(); // Формирование рейтинга

// Административные функции
permissions.isAdmin(); // Админские права
permissions.canManageUsers(); // Управление пользователями
permissions.canManageRoles(); // Управление ролями

// Доступ к данным
permissions.canAccessSchool(schoolId); // Доступ к конкретной школе
permissions.canManageAllSchools(); // Доступ ко всем школам
```

## 📁 Структура навигации

Меню администрирования теперь динамически формируется на основе разрешений:

- **Пользователи**: Только для админов
- **Роли**: Только для админов
- **Аналитика**: Для тех, у кого есть `can_view_analytics`
- **Отчеты**: Для тех, у кого есть `can_view_reports`
- **Настройки**: Только для админов

## ✅ Особенности реализации

1. **API-first подход**: Все разрешения берутся из API
2. **Типобезопасность**: Полная типизация с TypeScript
3. **Реактивность**: Автоматическое обновление UI при изменении разрешений
4. **Производительность**: Минимальные пересчеты благодаря хукам
5. **Расширяемость**: Легко добавлять новые роли и разрешения

## 🚀 Готовые компоненты

- `AdminGuard` - защита админских функций
- `AnalyticsGuard` - доступ к аналитике
- `ReportsGuard` - доступ к отчетам
- `DataInputGuard` - доступ к вводу данных
- `SchoolDataGuard` - доступ к данным конкретной школы
